## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 재료를 선택하거나 선택하지 않는 모든 경우를 탐색하는 DFS/완전탐색을 생각했습니다.
- 핵심 아이디어:
    - 각 재료는 한 번만 사용할 수 있으므로 전형적인 0/1 Knapsack 문제로 변환할 수 있습니다.
    - `dp[j]`를 "칼로리 j 이하에서 얻을 수 있는 최대 맛 점수"로 정의하여 해결했습니다.
- 사용한 알고리즘:
    - 1차원 DP (배낭 문제)
- 시간 복잡도 예상:
    - `O(N × L)`
- 공간 복잡도 예상:
    - `O(L)`

---

## 🚀 구현

```java
import java.util.*;

public class Solution {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {

            int N = sc.nextInt();
            int L = sc.nextInt();

            int[] dp = new int[L+1];

            for(int i=0 ; i<N ; i++) {

                int score = sc.nextInt();
                int cal = sc.nextInt();

                for(int j=L ; j>=cal ; j--) {
                    dp[j] = Math.max(dp[j], dp[j-cal] + score);
                }
            }

            System.out.println("#" + t + " " + dp[L]);
        }
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 테스트케이스 수를 입력받습니다.
    2. 재료 개수 `N`과 제한 칼로리 `L`을 입력받습니다.
    3. `dp[j]`를 칼로리 `j` 이하에서 만들 수 있는 최대 맛 점수로 정의합니다.
    4. 각 재료에 대해:
        - 현재 재료를 사용하지 않는 경우: `dp[j]`
        - 현재 재료를 사용하는 경우: `dp[j-cal] + score`
        - 둘 중 큰 값을 저장합니다.
    5. 최종적으로 `dp[L]`을 출력합니다.
- 중요한 포인트:
    - 반드시 뒤에서부터 순회해야 합니다.
    - 앞에서부터 순회하면 같은 재료를 여러 번 사용하는 문제가 발생합니다.
- 실수했던 부분:
    - 처음에는 `dp[j]` 값들이 모두 같은 값으로 채워지는 것으로 생각했지만, 실제로는 `dp[j-cal]` 값이 위치마다 달라지기 때문에 서로 다른 최적값이 저장됩니다.

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - DP 배열을 앞에서부터 갱신하면 같은 재료가 중복 사용되는 문제가 발생했습니다.
- 반례:

```
재료:점수 5, 칼로리 3앞에서부터 갱신 시:dp[3] = 5dp[6] = dp[3] + 5 = 10같은 재료를 두 번 사용한 잘못된 결과 발생
```

- 디버깅 과정:
    - `j`를 감소시키며 순회하도록 수정하여 문제를 해결했습니다.
