## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 배열을 Queue처럼 앞에서 빼고 뒤에 넣는 방식으로 구현
- 핵심 아이디어:
    - 맨 앞 숫자를 꺼내 감소시키고 맨 뒤로 이동
    - 감소값은 1~5 반복
    - 마지막 값이 0 이하가 되면 종료
- 사용한 알고리즘:
    - 시뮬레이션
    - Queue 구조 구현
- 시간 복잡도 예상:
    - O(N × 반복횟수)
    - 실제 N=8 고정이라 매우 작음
- 공간 복잡도 예상:
    - O(1)

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        for(int t=1 ; t<=10 ; t++) {
        	int test_case = sc.nextInt();
        	int[] arr = new int[8];
        	
        	for(int i=0 ; i<8 ; i++) {
        		arr[i] = sc.nextInt();
        	}
        	
        	while(true) {
        		
        		for(int i=1 ; i<=5 ; i++) {
        			int tmp = arr[0];
        			
        			for(int j=0 ; j<7 ; j++) {
        				arr[j] = arr[j+1];
        			}
        			arr[7] = tmp - i;
        			
        			if(arr[7] <= 0) {
        				arr[7] = 0;
        				break;
        			}
        		}
        		
        		if(arr[7] <= 0) {
    				arr[7] = 0;
    				break;
        		}
        	}
        	
        	System.out.print("#" + test_case + " ");
        	
        	for(int i=0 ; i<8 ; i++) {
        		System.out.print(arr[i] + " ");
        	}
        	
        	System.out.println();
        }
        
    	sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 8개의 숫자를 입력받음
    2. while문으로 암호 생성 반복
    3. 감소값 1~5를 순서대로 적용
    4. 맨 앞 숫자를 꺼내 감소 후 뒤에 삽입
    5. 감소 결과가 0 이하라면 0으로 저장 후 종료
    6. 최종 배열 출력
- 중요한 포인트:
    - Queue를 직접 구현하기 위해 배열을 왼쪽으로 한 칸씩 이동
    - 감소값은 반드시 1→2→3→4→5 순환
    - 0 이하가 되는 즉시 종료해야 함
- 실수했던 부분:
    - 종료 조건을 내부 반복문과 외부 반복문 둘 다 체크해야 함
    - 감소 후 바로 0 처리 필요

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - 보통 많이 발생하는 실수는 감소 후 음수를 그대로 저장하는 경우
    - 또는 감소값 순환을 제대로 처리하지 않는 경우
- 반례:

```
1 2 3 4 5 6 7 1
```

- 디버깅 과정:
    - 감소 후 배열 이동이 올바르게 되는지 확인
    - 마지막 값이 0이 되는 순간 즉시 종료되는지 체크

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
	- 현재는 배열을 매번 한 칸씩 당기므로 비효율적
	- Queue 자료구조 사용 시 코드가 훨씬 간단해짐
- 다른 사람 풀이 참고:
	- 대부분 `Queue<Integer>` 사용
	- poll() $\rightarrow$ 감소 $\rightarrow$ offer() 구조 사용
  
- 개선된 코드:
  
```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        for(int t=1 ; t<=10 ; t++) {
            int tc = sc.nextInt();

            Queue<Integer> q = new LinkedList<>();

            for(int i=0 ; i<8 ; i++) {
                q.offer(sc.nextInt());
            }

            int minus = 1;

            while(true) {
                int num = q.poll() - minus;

                if(num <= 0) {
                    q.offer(0);
                    break;
                }

                q.offer(num);

                minus++;

                if(minus == 6) {
                    minus = 1;
                }
            }

            System.out.print("#" + tc + " ");

            while(!q.isEmpty()) {
                System.out.print(q.poll() + " ");
            }

            System.out.println();
        }

        sc.close();
    }
}
```

## 📚 배운 점

- 새로 알게 된 개념:
    - Queue 구조는 순환 처리 문제에서 매우 유용
    - 배열로 Queue를 구현하면 불필요한 이동 연산 발생
- 기억해야 할 패턴:
    - `poll → 연산 → offer`
    - 종료 조건 발생 시 즉시 break
- 다음에 적용할 점:
    - Queue 성격 문제는 배열보다 Queue 자료구조 우선 고려
    - 반복 감소 패턴은 modulo 또는 순환 변수 사용 가능
