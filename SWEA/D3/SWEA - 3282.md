## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 물건을 선택하거나 선택하지 않는 경우를 모두 고려해야 하므로 DP를 사용했습니다.
- 핵심 아이디어:
    - `dp[j]`를 "부피가 j일 때 얻을 수 있는 최대 가치"로 정의했습니다.
    - 각 물건마다 뒤에서부터 순회하며 현재 물건을 한 번만 사용하도록 처리했습니다.
- 사용한 알고리즘:
    - 0/1 배낭 문제(Dynamic Programming)
- 시간 복잡도 예상:
    - `O(N × K)`
- 공간 복잡도 예상:
    - `O(K)`

---

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {
        	
        	int n = sc.nextInt();
        	int k = sc.nextInt();
        	int[] dp = new int[k+1];
        	
        	for(int i=0 ; i<n ; i++) {
        		int v = sc.nextInt();
        		int c = sc.nextInt();
        		
        		for(int j=k ; j>=v ; j--) {
        			dp[j] = Math.max(dp[j], dp[j-v] + c);
        		}
        	}
        	
        	System.out.println("#" + t + " " + dp[k]);
        }
        
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 테스트케이스 수 입력
    2. 물건 개수 `n`, 가방 최대 부피 `k` 입력
    3. `dp[j]`에 해당 부피에서 가능한 최대 가치 저장
    4. 각 물건에 대해:
        - 부피 `v`, 가치 `c` 입력
        - `j=k ~ v`까지 역순 순회
        - 현재 물건을 넣는 경우와 넣지 않는 경우 비교
    5. 최종적으로 `dp[k]` 출력
- 중요한 포인트:
    - 반드시 뒤에서부터 순회해야 같은 물건이 여러 번 사용되지 않습니다.
    - 정순으로 돌면 같은 물건을 중복 선택하는 문제가 발생합니다.
- 실수했던 부분:
    - DP 배열이 처음엔 모두 0이라서 초반에는 여러 칸이 같은 값으로 채워질 수 있다는 점이 헷갈릴 수 있습니다.
    - 하지만 이후 다른 물건들이 추가되면서 값이 계속 갱신됩니다.

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - 만약 `for(int j=v ; j<=k ; j++)`처럼 정순 순회를 하면 같은 물건을 여러 번 사용하는 상태가 됩니다.
- 반례:

```
물건:
부피 2 가치 10

정순 순회 시:dp[2] = 10dp[4] = dp[2] + 10 = 20

→ 같은 물건을 2번 사용한 결과
```

- 디버깅 과정:
    - dp 배열 변화 과정을 직접 출력해보면서 역순 순회의 필요성을 확인했습니다.

## 📚 배운 점

- 새로 알게 된 개념:
    - 0/1 Knapsack에서는 역순 순회가 핵심입니다.
- 기억해야 할 패턴:
    - "물건을 한 번만 사용" → 역순
    - "물건을 여러 번 사용 가능" → 정순
- 다음에 적용할 점:
    - DP 정의를 먼저 명확히 하고,
    - 상태 전이가 중복 사용을 허용하는지 여부를 항상 확인해야 합니다.
