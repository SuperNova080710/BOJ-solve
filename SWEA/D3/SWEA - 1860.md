## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 시간을 0초부터 마지막 손님 도착 시간까지 순회하면서 붕어빵 개수를 직접 관리하는 시뮬레이션 방식으로 접근했습니다.
- 핵심 아이디어:
    - 특정 시간까지 만든 붕어빵 개수가 해당 시간까지 도착한 손님 수보다 적으면 불가능합니다.
    - 손님 도착 시간을 정렬하면 순서대로 검증할 수 있습니다.
- 사용한 알고리즘:
    - 정렬
    - 시뮬레이션
- 시간 복잡도 예상:
    - 기존 구현: `O(maxTime)`
    - 개선 구현: `O(N log N)`
- 공간 복잡도 예상:
    - `O(N)`

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {
        	int n = sc.nextInt();
        	int m = sc.nextInt();
        	int k = sc.nextInt();
        	int[] num = new int[n];
        	int cnt=0;
        	int index=0;
        	String result = "Possible";

        	for(int i=0 ; i<n ; i++) {
        		num[i] = sc.nextInt();
        	}

        	Arrays.sort(num);

        	for(int i=0 ; i<=num[num.length-1] ; i++) {

        		if(i%m==0 && i!=0) {
        			cnt+=k;
        		}

        		while(index < n && num[index]==i) {

        			if(cnt>0) {
        				cnt--;
        			} else {
        				result = "Impossible";
        				break;
        			}

        			index++;
        		}

        		if(result.equals("Impossible")) {
        			break;
        		}
        	}

        	System.out.println("#" + t + " " + result);

        }

    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 손님 도착 시간을 입력받아 정렬합니다.
    2. 시간을 0초부터 마지막 손님 도착 시간까지 증가시키며 검사합니다.
    3. `m`초마다 붕어빵 `k`개를 추가 생산합니다.
    4. 현재 시간에 도착한 손님들을 모두 처리합니다.
    5. 붕어빵이 부족하면 `"Impossible"`을 출력합니다.
- 중요한 포인트:
    - 같은 시간에 여러 명이 올 수 있으므로 `if`가 아니라 `while`로 처리해야 합니다.
    - `0초 손님`도 존재할 수 있으므로 시간을 `0부터` 순회해야 합니다.
- 실수했던 부분:
    - 처음에는 `0초 손님`을 고려하지 못했습니다.
    - 이후 수정했지만 같은 시간 손님 여러 명 처리 문제를 놓쳤습니다.

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - 같은 시간에 여러 명이 오는 경우 한 명만 처리했습니다.
- 반례:

```
1
2 2 2
4 4
```

- 디버깅 과정:
    - 정렬 후 같은 시간 값이 연속된다는 점을 확인했습니다.
    - `if(num[index] == i)` 대신 `while`로 수정하여 해결했습니다.

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
    - 시간을 전부 순회하지 않고 손님 도착 시간만 검사하는 방식이 더 효율적입니다.
- 다른 사람 풀이 참고:
    - `i번째 손님까지 필요한 붕어빵 개수`와 `현재까지 생산한 붕어빵 개수`를 비교하는 방식이 많이 사용되었습니다.
  
- 개선된 코드:
  
```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {

            int n = sc.nextInt();
            int m = sc.nextInt();
            int k = sc.nextInt();

            int[] arr = new int[n];

            for(int i=0 ; i<n ; i++) {
                arr[i] = sc.nextInt();
            }

            Arrays.sort(arr);

            String result = "Possible";

            for(int i=0 ; i<n ; i++) {

                int bread = (arr[i] / m) * k;

                if(bread < i + 1) {
                    result = "Impossible";
                    break;
                }
            }

            System.out.println("#" + t + " " + result);
        }
    }
}
```
