## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근: 
	- 입력할 때마다 현재 저장된 최대값과 비교한다.
	- 10개의 숫자를 모두 입력받으면 최종 최대값을 출력한다.
  
- 핵심 아이디어:
	- 숫자를 하나씩 입력받으면서 최대값을 갱신한다.
  
- 사용한 알고리즘:
	- 단순 반복문
	- 최대값 갱신
  
- 시간 복잡도 예상:
	- O(10T)=O(T)
  
- 공간 복잡도 예상:
	- O(1)

---

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();
        
        for(int t=1 ; t<=T ; t++) {
        	int max = 0;
        	
        	for(int i=0 ; i<10 ; i++) {
        		int n = sc.nextInt();
        		if(n > max) {
        			max = n;
        		}
        	}
        	
        	System.out.println("#" + t + " " + max);
        }

        
    	sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
	1. 테스트케이스 개수 `T`를 입력받는다.
	2. 각 테스트케이스마다 최대값 변수 `max`를 생성한다.
	3. 숫자 10개를 반복문으로 입력받는다.
	4. 입력받은 숫자가 현재 최대값보다 크면 갱신한다.
	5. 10개 입력이 끝나면 최대값을 출력한다.

- 중요한 포인트:
	- 배열을 사용하지 않고 바로 최대값을 비교했다.
	- 입력과 동시에 처리하여 메모리를 아낄 수 있다.
  
- 실수했던 부분:
	- `max`를 0으로 초기화하면 음수 입력에서는 문제가 생길 수 있다.
	- 하지만 해당 문제는 양수 범위라 정상 동작한다.

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
	- 최대값 초기화를 더 안전하게 할 수 있다.
  
- 다른 사람 풀이 참고:
	- Math.max()를 사용하는 방식도 자주 사용된다.
  
- 개선된 코드:
  
```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();
        
        for(int t=1 ; t<=T ; t++) {
        	int max = Integer.MIN_VALUE;
        	
        	for(int i=0 ; i<10 ; i++) {
        		int n = sc.nextInt();
        		max = Math.max(max, n);
        	}
        	
        	System.out.println("#" + t + " " + max);
        }

    	sc.close();
    }
}
```
