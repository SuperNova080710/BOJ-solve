## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
	- 0에서 시작해서 99까지 갈 수 있는지 탐색하면 된다고 생각했습니다.
	- 노드 간 연결 관계가 있으므로 그래프 탐색 문제라고 판단했습니다.
  
- 핵심 아이디어:
	- 그래프를 인접 리스트로 저장한 뒤 DFS로 탐색합니다.
	- 현재 노드에서 연결된 다음 노드들을 재귀적으로 탐색합니다.
	- 탐색 중 99에 도달하면 즉시 true를 반환합니다.
  
- 사용한 알고리즘:
	- DFS(깊이 우선 탐색)
  
- 시간 복잡도 예상:
	- O(V + E)
  
- 공간 복잡도 예상:
	- O(V)

## 🚀 구현

```java
import java.util.*;

public class Main {
	static List<Integer>[] graph;
	static boolean[] visited;
	
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        for(int t=1 ; t<=10 ; t++) {
        	int tc = sc.nextInt();
        	int N = sc.nextInt();
        	
        	graph = new ArrayList[100];
        	visited = new boolean[100];
        	
        	for(int i=0 ; i<100 ; i++) {
        		graph[i] = new ArrayList<Integer>();
        	}
        	
        	for(int i=0 ; i<N ; i++) {
        		int from = sc.nextInt();
        		int to = sc.nextInt();
        		graph[from].add(to);
        	}
        	
        	boolean result = dfs(0);
        	
        	System.out.println("#" + t + " " + (result ? 1:0));
        	
        	
        }
        
        
    }
    
    static boolean dfs(int node) {
    	if(node == 99) return true;
    	
    	visited[node] = true;
    	
    	for(int next : graph[node]) {
    		if(!visited[next]) {
    			if(dfs(next)) return true;
    		}
    	}
    	
    	return false;
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
	1. 그래프를 인접 리스트 형태로 저장합니다.
	2. 시작 노드인 0부터 DFS 탐색을 시작합니다.
	3. 방문한 노드는 visited 배열로 체크합니다.
	4. 연결된 다음 노드들을 재귀적으로 탐색합니다.
	5. 탐색 도중 99에 도달하면 true를 반환합니다.
	6. 모든 경로를 탐색했는데도 99에 도달하지 못하면 false를 반환합니다.

- 중요한 포인트:
	- `visited`를 사용하지 않으면 사이클 때문에 무한 재귀가 발생할 수 있습니다.
	- `if (dfs(next)) return true;`
	    - 이 부분이 핵심입니다.
	    - 하나의 경로라도 성공하면 즉시 탐색을 종료합니다.
  
- 실수했던 부분:
	- 테스트케이스마다 graph와 visited를 초기화하지 않으면 이전 값이 남아 영향을 줄 수 있습니다.

## ❗ 문제점 / 시행착오

- 틀린 이유:
	- 방문 체크를 하지 않으면 이미 방문한 노드를 계속 탐색하게 됩니다.
  
- 반례:

```
0 -> 1  
1 -> 0
```
  
- 디버깅 과정:
	- DFS 흐름을 따라가며 현재 노드와 다음 노드를 출력해보면서 확인했습니다.

## 📚 배운 점

- 새로 알게 된 개념:
	- DFS는 한 경로를 끝까지 탐색한 뒤 돌아오는 방식입니다.
	- 재귀 호출의 반환값을 이용해 탐색 성공 여부를 전달할 수 있습니다.
  
- 기억해야 할 패턴:
  
```java
if (!visited[next]) {
    if (dfs(next)) {
        return true;
    }
}
```
  
- 다음에 적용할 점:
	- 그래프 탐색 문제에서 “도달 가능 여부”가 보이면 DFS/BFS를 먼저 떠올리기
