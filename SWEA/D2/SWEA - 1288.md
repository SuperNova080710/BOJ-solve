## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 입력받은 숫자의 배수를 계속 만들어가며 등장한 숫자를 확인한다.
    - 0~9까지 모든 숫자가 등장하면 종료한다.
- 핵심 아이디어:
    - 각 배수를 문자열로 변환한 뒤 자릿수를 하나씩 확인한다.
    - 숫자 등장 여부를 배열로 관리한다.
- 사용한 알고리즘:
    - 구현
    - 시뮬레이션
- 시간 복잡도 예상:
    - `O(K × D)`
    - K: 배수 반복 횟수
    - D: 숫자의 자릿수
- 공간 복잡도 예상:
    - `O(1)`

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        int T = sc.nextInt();
        
        int[] count = new int[10];
        int[] countChecked = new int[10];

        for(int t=1 ; t<=T ; t++) {

            // 배열 초기화
            Arrays.fill(count, 0);
            Arrays.fill(countChecked, 1);

            int n = sc.nextInt();

            String nString;

            int repeat = 1;
            int multiply = n;

            while(!Arrays.equals(count, countChecked)) {

                multiply = n * repeat;

                // 숫자를 문자열로 변환
                nString = Integer.toString(multiply);

                // 각 자리 숫자 확인
                for(int i=0 ; i<nString.length() ; i++) {

                    int digit = nString.charAt(i) - '0';

                    // 처음 등장한 숫자만 체크
                    if(count[digit] == 1) {
                        continue;
                    } else {
                        count[digit] = 1;
                    }
                }

                repeat++;
            }

            System.out.println("#" + t + " " + multiply);
        }

        sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 테스트케이스 입력.
    2. 숫자 등장 여부 배열 초기화.
    3. `n`의 배수를 계속 생성.
    4. 문자열로 변환 후 각 자리 숫자 확인.
    5. 등장한 숫자를 배열에 기록.
    6. 0~9까지 모두 등장하면 종료.
- 중요한 포인트:
    - `Arrays.fill()`로 테스트케이스마다 배열 초기화 필요.
    - `charAt(i) - '0'`으로 문자 숫자를 정수로 변환.
    - 종료 조건은 `Arrays.equals()` 사용.
- 실수했던 부분:
    - 초기에는 배열 초기화를 반복문 밖에서 수행했음.
    - 출력값으로 반복 횟수를 출력하고 있었음.
    - `multiply` 갱신 순서가 잘못되어 있었음.

## ❗ 문제점 / 시행착오

- 틀린 이유:
	- 테스트케이스 간 배열 상태가 유지됨.
	- 출력값이 문제 요구사항과 달랐음.
	- 배수 계산 순서가 꼬여 있었음.
  
- 반례:

```
2
1
2
```

첫 번째 테스트케이스 이후 배열이 초기화되지 않으면 오답 발생 가능.

- 디버깅 과정:
	- 배열 상태를 출력해보며 이전 값이 유지되는 문제 확인.
	- 반복문 내부 흐름을 추적하며 `multiply` 갱신 위치 수정.
	- 출력값이 실제 마지막 수인지 검토.

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
    - 문자열 변환 대신 `% 10`, `/ 10` 사용 가능.
    - `Arrays.equals()` 대신 숫자 개수 카운트 방식 사용 가능.
- 다른 사람 풀이 참고:
    - `boolean[]` 배열을 사용하는 방식이 더 직관적.
    - 비트마스킹으로도 구현 가능.
  
- 개선된 코드:
  
```java
boolean[] visited = new boolean[10];
int checkedCount = 0;

while(checkedCount < 10) {

    int temp = result;

    while(temp > 0) {

        int digit = temp % 10;

        if(!visited[digit]) {
            visited[digit] = true;
            checkedCount++;
        }

        temp /= 10;
    }
}
```
