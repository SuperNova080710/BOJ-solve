## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 두 회사의 수도 요금을 각각 계산한 뒤 더 저렴한 값을 출력하면 된다고 판단했습니다.
- 핵심 아이디어:
    - A회사:
        - 사용량 × 리터당 요금
    - B회사:
        - 기본 요금 + 초과 사용량 × 추가 요금
    - 두 값을 비교하여 최소값 선택
- 사용한 알고리즘:
    - 단순 구현
    - 조건문(if)
- 시간 복잡도 예상:
    - O(1)
- 공간 복잡도 예상:
    - O(1)

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();
        
        for(int t=1 ; t<=T ; t++) {
        	int p,q,r,s,w;
        	int result=0;
        	
        	p = sc.nextInt();
        	q = sc.nextInt();
        	r = sc.nextInt();
        	s = sc.nextInt();
        	w = sc.nextInt();
        	
        	int a = p*w;
        	int b;
        	
        	if(w<=r) {
        		b = q;
        	} else {
        		b = q + ((w-r)*s);
        	}
        	
        	if(a>=b) {
        		result = b;
        	} else {
        		result = a;
        	}
        	
        	System.out.println("#" + t + " " + result);
        }
        
    	sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 테스트 케이스 개수 입력
    2. 각 회사의 요금 정책 입력
    3. A회사 요금 계산
    4. B회사 요금 계산
        - 기준 사용량 이하이면 기본 요금만 부과
        - 초과 시 추가 요금 계산
    5. 두 회사 요금 비교 후 더 작은 값 출력
- 중요한 포인트:
    - B회사는 `w <= r` 조건일 때 추가 요금이 발생하지 않는다는 점이 핵심입니다.
    - 초과 사용량은 `(w-r)`로 계산합니다.
- 실수했던 부분:
    - B회사 계산 시 초과 사용량 계산을 빼먹거나,
    - `w < r`로 작성해서 경계값 처리를 틀리기 쉽습니다.

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - 특별한 오류 없이 구현 가능한 문제입니다.
    - 다만 조건 경계값 처리 실수가 자주 발생합니다.
- 반례:

```
110 100 20 3 20
```

- 설명:
    - 사용량이 정확히 기준값 `r`과 같은 경우
    - B회사는 추가 요금 없이 `q`만 나와야 합니다.
- 디버깅 과정:
    - `w<=r` 조건이 정확히 들어갔는지 확인
    - 초과 요금 계산 시 괄호 확인

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
    - `Math.min()`을 사용하면 비교 코드가 더 간단해집니다.
- 다른 사람 풀이 참고:
    - 대부분 `Math.min(a,b)` 형태로 간결하게 구현합니다.
- 개선된 코드:
  
```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {

            int p = sc.nextInt();
            int q = sc.nextInt();
            int r = sc.nextInt();
            int s = sc.nextInt();
            int w = sc.nextInt();

            int a = p * w;

            int b = (w <= r)
                    ? q
                    : q + (w - r) * s;

            System.out.println("#" + t + " " + Math.min(a, b));
        }

        sc.close();
    }
}
```

## 📚 배운 점

- 새로 알게 된 개념:
    - 단순 구현 문제에서도 경계값 처리가 중요합니다.
    - `Math.min()`을 사용하면 코드 가독성이 좋아집니다.
- 기억해야 할 패턴:
