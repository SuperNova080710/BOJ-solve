## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
	- 큰 단위에서 한번씩 빼면서 반복(while 반복)
  
- 핵심 아이디어:
	- 각 화폐 단위마다 몫과 나머지 연산을 하여 한번에 개수 계산
  
- 사용한 알고리즘:
	- 그리디 알고리즘
  
- 시간 복잡도 예상:
	- 초기 풀이: O(N) (최악의 경우 10원 단위까지 반복 -> 매우 비효율)
	- 개선 풀이: O(1) (화폐 종류 8개로 고정)
  
- 공간 복잡도 예상:
	- O(1)

## 🚀 구현

```java
import java.util.*;

public class Main {
	public static void main(String[] args) throws Exception {
		Scanner sc = new Scanner(System.in);

		int t = sc.nextInt();
		
		for(int T = 1 ; T<=t ; T++) {
			int[] result = new int[8];
			int change = sc.nextInt();
			
			result[0] = change/50000;
			change %= 50000;
			
			result[1] = change/10000;
			change %= 10000;
			
			result[2] = change/5000;
			change %= 5000;
			
			result[3] = change/1000;
			change %= 1000;
			
			result[4] = change/500;
			change %= 500;
			
			result[5] = change/100;
			change %= 100;
			
			result[6] = change/50;
			change %= 50;
			
			result[7] = change/10;
			
			System.out.println("#" + T);
			for(int i=0 ; i<8 ; i++) {
				System.out.print(result[i] + " ");
			}
			System.out.println();
		}
		
		sc.close();
	}
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
	1. 테스트 케이스 입력
	2. 각 금액에 대해
	3. 큰 화폐 단위부터 나눗셈으로 개수 계산
	4. 나머지를 다음 계산에 사용
	5. 결과 출력

- 중요한 포인트:
	- - `change / 단위` → 해당 화폐 개수
	- `change % 단위` → 남은 금액
	- 화폐 단위가 **내림차순**이어야 함
  
- 실수했던 부분:
	- while문으로 반복하며 하나씩 차감 → 시간 초과 발생

## ❗ 문제점 / 시행착오

- 틀린 이유:
	- 반복문 기반 풀이에서 연산 횟수가 너무 많아짐
  
- 반례:
	- N = 1,000,000 같은 큰 값 → 10원 단위까지 반복 시 매우 비효율
  
- 디버깅 과정:
	- 시간 초과 원인 → 반복 횟수 확인
	- 나눗셈 방식으로 변경

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
	- 화폐 배열을 사용하여 반복문으로 처리하면 코드 간결화 가능
  
- 다른 사람 풀이 참고:
	- 배열 기반 반복 처리 (가독성 향상)
  
- 개선된 코드:
```java
import java.util.*;

public class Main {
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);
		
		int[] money = {50000, 10000, 5000, 1000, 500, 100, 50, 10};
		int t = sc.nextInt();
		
		for(int T = 1; T <= t; T++) {
			int change = sc.nextInt();
			int[] result = new int[8];
			
			for(int i = 0; i < 8; i++) {
				result[i] = change / money[i];
				change %= money[i];
			}
			
			System.out.println("#" + T);
			for(int r : result) {
				System.out.print(r + " ");
			}
			System.out.println();
		}
	}
}
```

## 📚 배운 점

- 새로 알게 된 개념:
	- 그리디 알고리즘은 **항상 최적해를 보장하는 조건**에서만 사용 가능
  
- 기억해야 할 패턴:
	- “최소 개수 / 최소 비용” → 큰 단위부터 처리하는 그리디 의심
  
- 다음에 적용할 점:
	- 반복문으로 하나씩 처리하는 방식보다 나눗셈으로 한 번에 처리할 수 있는지 먼저 판단
