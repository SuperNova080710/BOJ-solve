## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 길이가 더 짧은 배열을 긴 배열 위에서 한 칸씩 이동시키며 곱의 합을 계산
- 핵심 아이디어:
    - 두 배열 중 더 짧은 배열을 기준으로 슬라이딩하면서 최대 곱셈 합을 구함
- 사용한 알고리즘:
    - 브루트포스(완전 탐색)
- 시간 복잡도 예상:
    - O((N-M+1) × M)
- 공간 복잡도 예상:
    - O(N + M)

## 🚀 구현

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();
        
        for(int t=1 ; t<=T ; t++) {
        	int n = sc.nextInt();
        	int m = sc.nextInt();
        	int[] a = new int[n];
        	int[] b = new int[m];
        	int max = 0;

        	
        	if(n<m) {
        		int tmp = n;
        		n = m;
        		m = tmp;
        		
        		a = new int[m];
        		b = new int[n];
        		
        		for(int i=0 ; i<m ; i++) {
            		a[i] = sc.nextInt();
            	}
            	for(int j=0 ; j<n ; j++) {
            		b[j] = sc.nextInt();
            	}
        	} else {
        		
        		a = new int[n];
        		b = new int[m];
        		
        		for(int i=0 ; i<n ; i++) {
            		a[i] = sc.nextInt();
            	}
            	for(int j=0 ; j<m ; j++) {
            		b[j] = sc.nextInt();
            	}
        	}
        	
        	for(int i=0 ; i<n-m+1 ; i++) {
        		int multiply = 0;
        		
        		for(int j=0 ; j<m ; j++) {
        			if(a.length>b.length) {
        				multiply += a[j+i] * b[j];
        			} else {
        				multiply += a[j] * b[j+i];        				
        			}
        		}
        		
        		if(max<multiply) {
        			max = multiply;
        		}
        		
        	}
        	
        	System.out.println("#" + t + " " + max);
        	
        }
    	sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 두 배열의 길이와 값을 입력받음
    2. 항상 `a`가 더 긴 배열이 되도록 길이를 비교
    3. 짧은 배열을 긴 배열 위에서 이동시키며 곱의 합 계산
    4. 각 위치마다 최대값 갱신
    5. 최종 최대값 출력
- 중요한 포인트:
    - 배열 길이를 바꿀 때 입력 순서도 함께 고려해야 함
    - 슬라이딩 범위는 `n - m + 1`
    - 각 위치마다 곱셈 합을 새로 계산해야 함
- 실수했던 부분:
    - `max` 초기값을 0으로 설정하면 음수만 있는 경우 오답 가능성 존재
    - 문제 조건상 양수라 통과되지만 일반적인 코드라면 `Integer.MIN_VALUE`가 안전

## ❗ 문제점 / 시행착오

- 틀린 이유:
    - 배열 길이를 swap한 뒤 입력받는 부분이 직관적이지 않아 헷갈릴 수 있음
- 반례:

```
12 31 5-3 -1 -2
```

- 디버깅 과정:
    - 어떤 배열이 긴 배열인지 확인하면서 인덱스 범위를 계속 점검함
    - `a.length > b.length` 조건이 실제로 항상 참인지 확인 필요

## ✅ 개선 포인트 (리팩토링)

- 더 나은 방법:
    - 배열 자체를 다시 생성하지 않고 입력 후 참조만 교체하면 코드가 훨씬 단순해짐
- 다른 사람 풀이 참고:
    - 대부분 긴 배열/짧은 배열을 따로 변수로 두고 처리
    - 불필요한 조건문 제거
- 개선된 코드:
  
```java
import java.util.*;

public class Solution {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();

        for(int t=1 ; t<=T ; t++) {

            int n = sc.nextInt();
            int m = sc.nextInt();

            int[] arr1 = new int[n];
            int[] arr2 = new int[m];

            for(int i=0 ; i<n ; i++) {
                arr1[i] = sc.nextInt();
            }

            for(int i=0 ; i<m ; i++) {
                arr2[i] = sc.nextInt();
            }

            int max = Integer.MIN_VALUE;

            if(n >= m) {

                for(int i=0 ; i<=n-m ; i++) {

                    int sum = 0;

                    for(int j=0 ; j<m ; j++) {
                        sum += arr1[i+j] * arr2[j];
                    }

                    max = Math.max(max, sum);
                }

            } else {

                for(int i=0 ; i<=m-n ; i++) {

                    int sum = 0;

                    for(int j=0 ; j<n ; j++) {
                        sum += arr1[j] * arr2[i+j];
                    }

                    max = Math.max(max, sum);
                }
            }

            System.out.println("#" + t + " " + max);
        }

        sc.close();
    }
}
```

## 📚 배운 점

- 새로 알게 된 개념:
    - 두 배열 길이를 기준으로 슬라이딩 범위를 조절하는 방식
- 기억해야 할 패턴:
    - “짧은 배열을 긴 배열 위에서 이동”하는 브루트포스 패턴
- 다음에 적용할 점:
    - 배열 swap보다 참조 교체 방식이 코드 가독성이 좋음
    - 최대/최소 문제에서 초기값 설정 주의
