## 💡 접근 방법 (아이디어)

- 처음 떠올린 접근:
    - 학생들의 점수를 계산한 뒤 내림차순으로 정렬하면 배열의 인덱스 자체가 등수가 된다고 생각했습니다.
    - 정렬 이후에도 K번째 학생을 찾기 위해 학생 번호를 함께 저장했습니다.
- 핵심 아이디어:
    - 학생의 총점과 학생 번호를 이차원 배열에 저장합니다.
    - 총점을 기준으로 내림차순 정렬합니다.
    - 정렬 후 배열의 위치를 이용해 학점을 계산합니다.
    - K번째 학생의 위치를 찾아 해당 학점을 출력합니다.
- 사용한 알고리즘:
    - 정렬(Arrays.sort + Comparator)
    - 구현
- 시간 복잡도 예상:
    - O(N log N)
- 공간 복잡도 예상:
    - O(N)

## 🚀 구현

```java
package ssafy;
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int T = sc.nextInt();

        String[] score = {
                "A+", "A0", "A-",
                "B+", "B0", "B-",
                "C+", "C0", "C-",
                "D0"
        };

        for(int t=1 ; t<=T ; t++){

            int n = sc.nextInt();
            int k = sc.nextInt();

            double[][] studentScore = new double[n][2];

            for(int i=0 ; i<n ; i++) {

                int a = sc.nextInt();
                int b = sc.nextInt();
                int c = sc.nextInt();

                studentScore[i][0] =
                        (a * 0.35) +
                        (b * 0.45) +
                        (c * 0.2);

                studentScore[i][1] = i;
            }

            Arrays.sort(studentScore,
                    new Comparator<double[]>() {

                public int compare(double[] a, double[] b) {
                    return Double.compare(b[0], a[0]);
                }
            });

            for(int i=0 ; i<n ; i++) {

                if(studentScore[i][1] == k-1) {

                    System.out.println(
                            "#" + t + " " +
                            score[i / (n/10)]
                    );

                    break;
                }
            }
        }

        sc.close();
    }
}
```

## 🔍 풀이 설명

- 코드 흐름 설명:
    1. 학생들의 중간, 기말, 과제 점수를 입력받아 총점을 계산합니다.
    2. 총점과 학생 번호를 이차원 배열에 저장합니다.
    3. 총점을 기준으로 내림차순 정렬합니다.
    4. 정렬 후 K번째 학생의 위치를 찾습니다.
    5. 학생 위치를 기준으로 학점을 계산하여 출력합니다.
- 중요한 포인트:
    - 정렬 이후에도 특정 학생을 찾기 위해 학생 번호를 저장한 점
    - 배열 인덱스를 등수처럼 사용한 점
    - `i / (n/10)` 을 이용해 학점 구간을 계산한 점
- 실수했던 부분:
    - Comparator 구현 시 Java 버전에 따라 람다식이 동작하지 않았습니다.
    - `Double.compare()` 대신 단순 뺄셈 비교를 사용할 경우 실수 오차 문제가 발생할 수 있습니다.
